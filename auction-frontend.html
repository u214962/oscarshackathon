How prize encryption works:
1. Generate a key pair below (or in the RSA Helper tab).
2. Encrypt your prize text — only the ciphertext goes on-chain.
3. After the auction, the winner registers their public key.
4. You re-encrypt the prize for the winner; they decrypt it off-chain.
The prize is never in plaintext on-chain.

Prize encryption scheme
Why RSA for the prize?

Smart contracts are fully public — every byte on-chain is readable by anyone. RSA-OAEP ensures the prize ciphertext reveals nothing without the private key.

Double encryption flow:
  1. Organizer encrypts prize with their RSA public key → ciphertext on-chain
  2. Winner registers their RSA public key on-chain
  3. Organizer decrypts prize + re-encrypts with winner's key → delivers on-chain
  4. Winner decrypts with their private key → reads the prize

Security properties:
  Prize never in plaintext on-chain
  Organizer private key never touches blockchain
  Winner private key never shared
  Both public keys (n, e) on-chain for full auditability
Bid hash design
Single-layer keccak256:

bidHash = keccak256(abi.encodePacked(amount, password))

Pre-image resistance · Hiding · Binding · Front-running resistance

Known limitation: small amount ranges can be brute-forced. A strong password prevents this by adding entropy.
Protocol flow
BIDDING — commit hash + deposit. Bid hidden.
REVEAL  — submit (amount, password). Contract verifies hash. Mismatch forfeits deposit.
CLAIM   — top bidder claims. Second hash check = authentication.
ENDED   — refunds for all who revealed. Prize delivery begins.
Node.js CLI (production use)
# Organizer pre-deploy:
node prize-rsa.mjs generate-keys
node prize-rsa.mjs encrypt "MY_PRIZE"
node prize-rsa.mjs pubkey-components

# Winner post-auction:
node prize-rsa.mjs generate-winner-keys
node prize-rsa.mjs pubkey-components --winner

# Organizer delivers:
node prize-rsa.mjs decrypt "0xabc..."
node prize-rsa.mjs encrypt-for-winner "MY_PRIZE" <n> <e>

# Winner collects:
node prize-rsa.mjs decrypt-prize "0xdef..."
